# 자동차 조립 시스템 Spec

출처: `docs/[CRA_AI] Day2_1_Agentic Engineering.pdf` 20~24페이지

## 1. 차량 제조 순서

1. 자동차 타입을 선택한다.
2. 자동차에 들어갈 부품을 선택한다.
3. 완성된 차량을 테스트한다. (선택한 부품이 자동차 타입에 사용 가능한지 검사)

## 2. 제조 순서 1) 자동차 타입 선택

차량의 종류를 선택한다. 총 세 가지 타입을 제작할 수 있으며, 향후에 타입이 더 추가될 수 있다.

- Sedan (세단)
- SUV
- Truck (트럭)

## 3. 제조 순서 2) 자동차 조립

엔진, 제동장치(brake System), 조향장치(SteeringSystem)를 각각 선택한다.

| 부품 (Part)             | 선택 가능한 제조사                  |
|--------------------------|--------------------------------------|
| Engine                   | GM, TOYOTA, WIA                     |
| brake System (제동장치) | MANDO, Continental, Bosch           |
| SteeringSystem (조향장치)| Bosch, Mobis                        |

## 4. 제조 순서 3) 자동차 완성 가능 조합 확인

선택한 부품 조합이 유효한지 검사하는 제한조건.

### 제한조건 1

- 제동장치에 Bosch 제품을 사용했다면, 조향장치도 Bosch 제품을 사용해야 한다. (타사 제품과 호환되지 않는다.)

### 제한조건 2

- Continental은 Sedan용 제동장치를 만들지 않는다. → 세단에 Continental 제동장치 사용 불가
- 도요타(TOYOTA)는 SUV용 엔진을 만들지 않는다.
- WIA는 Truck용 엔진을 만들지 않는다.
- Mando는 Truck용 제동장치(brake System)를 만들지 않는다.

> 위 제한조건은 실제 사실과 다릅니다 (원본 자료 명시 사항. 실제 제조사별 생산 가능 여부와 무관한 테스트용 가상 조건).

## 5. 기존 Legacy System

- 이미 제작된 소스코드 존재
- 절차지향식(procedural)으로 구현된 소스코드
- 참고: https://github.com/mincoding-ai/assemblyCar

### 기존 시스템의 아쉬운 점

- 절차지향식 코드로, 유지보수가 어려운 구조
- 안전하지 않은 문법들이 사용됨
- 확장성이 고려되지 않음
- 유닛테스트가 없음
